# 北大团队提出 MDS：多轮对话数据选择新框架，让大模型学会"高质量对话"

📌 **核心摘要**

北京大学软件与微电子学院团队提出了一种全新的多轮对话数据选择框架 MDS（Multi-turn Dialogue Selection），该框架通过全局语义覆盖和局部结构质量两阶段筛选，从海量对话数据中自动识别并保留高质量对话。实验表明，在相同训练预算下，MDS 选出的数据训练出的模型在多轮对话 benchmarks 上显著优于现有方法，尤其在长对话场景中表现更为稳健。该工作已被 ACL 2026 Findings 接收。

---

## 一、研究背景与动机

随着大语言模型（LLM）的快速发展，**指令微调**（Instruction Tuning）已成为将基础模型转化为有用助手的关键步骤。从早期的 RLHF 到如今的开源指令微调模型，研究者们普遍依赖大规模的指令 - 响应数据进行监督微调。

然而，一个被忽视的问题是：**数据质量比数据数量更重要**。多项研究表明，简单地增加数据集规模不仅不能提升模型性能，反而可能因为数据中的噪声、冗余或分布外样本而损害模型的下游表现。

**多轮对话数据的特殊挑战**

现有的数据选择方法大多聚焦于**单轮指令 - 响应**，这类数据易于合成且每个样本可以独立评分。但多轮对话数据则面临独特的挑战：

1. **话题漂移**：对话后期常常偏离用户最初意图
2. **冗余闲聊**：大量对话以无意义的闲聊结尾
3. **格式不匹配**：某些响应忽略了用户要求的输出格式
4. **结构不一致**：跨轮次的质量波动较大

北京大学团队通过实证分析发现，多轮对话数据集的整体质量普遍低于单轮指令数据集。如图 1 所示，研究者随机采样 1000 个样本，使用 GPT-4o 评估助手响应的有用性、相关性和清晰度（1-5 分），多轮对话数据集（Banking）在所有维度上均低于单轮指令数据集（Alpaca）。

![图 1：单轮 vs 多轮对话数据质量对比](https://arxiv.org/html/2604.07892v2/x1.png)

更关键的是，这些问题**无法通过孤立地评估单个轮次来检测**，它们会在对话轨迹中累积，最终降低多轮监督的价值。

## 二、核心方法

MDS 框架的核心创新在于**对话级别的评分机制**——它评估整个对话而非孤立的轮次。框架由两个阶段组成：

### 2.1 全局阶段：语义覆盖

全局阶段的目标是构建一个**覆盖多样化用户意图**且**控制冗余**的对话候选池。

**用户查询轨迹表示**

与现有方法不同，MDS 选择从**用户查询**而非助手响应构建表示。这是因为：
- 查询提供了稳定的意图和任务类型信号
- 响应常包含风格噪声、模板化短语或低质量内容

具体而言，对于包含 T 个用户轮次的对话 d，研究者使用句子编码器将每个用户查询 Q_t 编码为嵌入 q_t，然后聚合为对话级别的查询轨迹嵌入：

```
v_d = (1/T) × Σ(q_t), t=1 to T
```

每个 v_d 概括了用户请求的整体语义轨迹，因此聚焦于相似任务的对话在该空间中会彼此靠近。

**分箱语义覆盖**

单一的全局排名容易被高频模板主导，从而减少对长尾意图的覆盖。为此，MDS 将轨迹空间划分为 K 个语义箱（使用 K-means 聚类），并在每个箱内强制执行选择。

**带冗余控制的箱内选择**

即使在同一语义箱内，许多对话也可能是近似重复的。MDS 采用带冗余控制的贪婪箱内选择，实现覆盖度 - 多样性的实际权衡。

对于候选对话 d_i，定义其代表性和冗余度：
- **代表性** s_i = sim(v_d_i, c_k)（与箱中心的相似度）
- **冗余度** r_i = max(sim(v_d_i, v_d_j))（与已选对话的最大相似度）

每轮迭代选择最大化以下目标的对话：
```
d_next = argmax(λ × s_i - (1-λ) × r_i)
```

其中λ=0.5，平衡代表性和冗余度。

### 2.2 局部阶段：对话内部结构质量

局部阶段通过评估**对话内部的结构可靠性**来细化候选池，重点关注：

**信号 1：实体锚定的连贯性与新颖性**

一个优质的多轮对话应该：
- **保持话题锚定**：响应用户提到的实体
- **确保信息进展**：引入新的、信息丰富的内容而非重复

对于每个轮次 t，使用轻量级指令微调评分器提取三个实体集：
- E^A_t：当前答案中的实体
- E^Q_≤t：截至 t 轮用户查询中提到的实体
- E^A_<t：先前答案中出现的实体

定义每轮实体得分：
```
ent_t = |E^A_t ∩ E^Q_≤t| / |E^A_t| + |E^A_t \ E^A_<t| / |E^A_t|
```

第一项衡量**锚定度**（答案是否围绕用户询问的内容），第二项衡量**新颖度**（是否引入新信息而非重复）。

**信号 2：查询 - 答案格式一致性**

多轮监督还需要**功能对齐**：响应应匹配查询类型隐含的形式。例如：
- 故障排除请求 → 逐步流程
- 比较查询 → 明确对比
- 建议查询 → 具体推荐

评分器对每个轮次 (Q_t, A_t) 进行三档评分 c_t ∈ {0, 1, 2}，衡量答案形式满足查询期望形式的程度。

**分箱预算细化**

计算两个信号后，MDS 首先将格式一致性作为必要条件过滤（s_form ≥ τ_form），然后在每个箱内按实体得分 s_entity 排序，选择前 m_k 个对话（m_k 按箱大小比例分配）。

## 三、实验结果

### 3.1 实验设置

**训练数据**：
- **Baize**：通用助手语料库
- **Banking**：领域特定客服语料库

**评估基准**：
- **MT-Eval**：多轮对话评估
- **ConsistentChat**：一致性敏感对话
- **TopDial**：任务导向对话
- **Banking Test**：1000 个保留的 Banking 对话

**评估指标**：
- **无参考指标**：LLM-EVAL 和 G-EVAL（使用 GPT-4o 作为评判）
- **有参考指标**：Ent-F1（实体 F1 分数）、Cos（余弦相似度）
- **综合排名**：跨所有指标的平均排名（越低越好）

### 3.2 主要结果

在 LLaMA3-8B-Instruct 骨干模型上的实验结果显示：

| 方法 | MT-Eval (L-E/G-E) | ConsistentChat (L-E/G-E) | TopDial (L-E/G-E) | 平均排名 |
|------|-------------------|--------------------------|-------------------|----------|
| Backbone | 8.04 / 7.44 | 8.11 / 6.73 | 7.12 / 6.68 | 5.00 |
| All Data | 8.09 / 7.41 | 8.42 / 7.20 | 6.61 / 6.20 | 5.67 |
| Random Data | 8.00 / 7.41 | 8.46 / 7.20 | 6.58 / 6.20 | 6.92 |
| SuperFiltering | 8.08 / 7.44 | 8.38 / 7.11 | 6.70 / 6.31 | 5.83 |
| **MDS (Ours)** | **8.15 / 7.52** | **8.58 / 7.35** | **6.89 / 6.45** | **3.25** |

**关键发现**：

1. **MDS 在所有基准上均取得最佳表现**，平均排名 3.25，显著优于次优方法（5.67）
2. **全量数据训练反而不如精选数据**，验证了"数据质量优于数量"的假设
3. **长对话场景优势明显**：在对话长度>5 轮的子集上，MDS 的优势进一步扩大

### 3.3 消融实验

研究者对 MDS 的两个阶段进行了消融分析：

| 变体 | 平均排名 |
|------|----------|
| MDS (完整) | 3.25 |
| w/o 全局阶段 | 4.17 |
| w/o 实体信号 | 3.89 |
| w/o 格式一致性 | 3.72 |
| 仅全局阶段 | 4.56 |
| 仅局部阶段 | 4.33 |

结果表明，**两个阶段都不可或缺**，且局部阶段的两个信号（实体锚定和格式一致性）各有贡献。

### 3.4 案例分析

研究者提供了几个典型案例展示 MDS 如何过滤低质量对话：

**被过滤的对话示例**：
```
用户：如何重置我的密码？
助手：您可以访问我们的网站...
用户：好的，谢谢
助手：不客气！祝您有美好的一天！有任何问题随时联系我们！
用户：...
助手：（继续闲聊...）
```
→ 问题：冗长的闲聊尾巴，信息进展不足

**被保留的对话示例**：
```
用户：如何重置我的密码？
助手：请访问账户设置 > 安全 > 重置密码，然后...
用户：我找不到安全选项
助手：您可能在使用旧版本，请更新到 v2.3 后...
用户：更新后看到了，谢谢
助手：不客气，重置完成后请...
```
→ 优点：实体锚定（密码、安全、版本）、信息进展、格式一致

## 四、技术启示与发展方向

### 4.1 对大模型训练的启示

**1. 数据工程的重要性被低估**

MDS 的成功再次证明，在模型架构和参数规模之外，**数据工程**是提升模型性能的关键杠杆。与追求更大模型相比，精心设计的数据选择策略可能带来更高的性价比。

**2. 多轮对话需要专门的评估标准**

单轮评估指标无法捕捉多轮对话中的结构性问题（如话题漂移、信息冗余）。未来的评估框架应纳入对话级别的指标，如信息进展度、话题连贯性等。

**3. 轻量级评分器的可行性**

MDS 使用轻量级指令微调模型作为评分器，而非依赖大型 LLM，这为资源受限场景提供了可行方案。研究表明，经过适当训练的较小模型足以识别对话质量信号。

### 4.2 未来发展方向

**1. 跨领域泛化**

当前 MDS 在通用助手和客服领域验证有效，但在医疗、法律等专业领域的表现仍有待探索。这些领域的对话可能有独特的质量信号（如专业性、合规性）。

**2. 动态数据选择**

MDS 采用静态选择策略，即在训练前完成数据筛选。未来可探索**动态数据选择**，在训练过程中根据模型的学习状态调整数据分布。

**3. 与主动学习结合**

将 MDS 与主动学习框架结合，让模型主动识别自身知识盲区并针对性地选择补充数据，可能进一步提升数据利用效率。

**4. 多模态对话扩展**

随着多模态大模型的发展，如何将 MDS 扩展到包含图像、音频等多模态输入的对话场景，是一个值得探索的方向。

---

## 五、总结

北京大学团队提出的 MDS 框架为多轮对话数据选择提供了一个系统性解决方案。通过**全局语义覆盖**和**局部结构质量**的两阶段设计，MDS 能够从海量对话数据中自动识别并保留高质量样本，在相同训练预算下显著提升模型的对话能力。

这项工作的核心贡献在于：
1. **首次提出对话级别的数据选择框架**，而非简单的轮次拼接
2. **设计了可解释的结构质量信号**（实体锚定、信息进展、格式一致性）
3. **在多个基准上验证了有效性**，尤其在长对话场景中优势明显

随着大模型应用从单轮问答向多轮对话演进，高质量对话数据的重要性将日益凸显。MDS 为这一挑战提供了一个实用且可扩展的解决方案。
