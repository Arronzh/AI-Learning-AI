# LLM 训练数据排序的"最优解"：微软提出 SAW 方法，系统性破解数据组织迷局

📌 **核心摘要**：本文系统性地探索了数据组织对 LLM 训练的影响，提出了四条关键准则——边界锐化、循环调度、课程连续性和局部多样性，并据此设计出 STR 和 SAW 两种排序策略。实验证明，SAW 方法在预训练和 SFT 阶段均可显著提升模型性能，且无需额外计算开销。

---

## 一、研究背景与动机

大语言模型（LLM）的训练效率高度依赖数据策展（data curation）的质量。过去几年，学术界和工业界在数据获取、混合、合成、去重、过滤和选择等方面投入了大量精力，用极其昂贵的计算资源为每个样本计算质量分数、难度分数、教育价值等指标。然而，这些精心计算的分数在完成数据选择后往往就被废弃了——同一个样本分数只能"用一次"。

与此同时，当前 LLM 的训练范式正在发生变化。由于训练语料规模巨大（动辄数万亿 token），大多数现代 LLM 只在一到数个 epoch 上完成训练。在这种"单次遍历"的范式下，**数据呈现给模型的顺序——而非仅仅是哪些数据被选中——已经成为影响学习轨迹的一阶因素。** 这个看似简单的问题，在学界却一直缺乏系统性的研究。

现有的方法如课程学习（Curriculum Learning, CL），简单地将数据按分数从低到高排列，它在小模型和计算机视觉任务上效果不错，但在 LLM 训练中存在明显缺陷：当模型在高分数据上训练太久后，会逐渐遗忘早期简单样本中学到的知识。而 DELT（Dai et al., 2025a）虽然引入了折叠（folding）机制来回顾不同分数段的数据，但它和 CL 一样，都没有系统性地研究数据排序背后的指导性原则。

微软亚洲研究院联合南洋理工大学、香港科技大学的研究团队，正是瞄准了这一空白。他们的论文《Demystifying Data Organization for Enhanced LLM Training》被 ACL 2026 Main Conference 接收，首次系统性地探索了数据组织对 LLM 训练的影响，提出了四条关键准则，并据此设计出 STR 和 SAW 两种新的排序方法。

值得注意的是，这项工作的核心创新在于**复用**已有的样本级分数来指导数据排序，而不是重新计算分数。这意味着它的计算成本几乎为零——那些花费了大量算力计算出来的样本分数，现在可以发挥"第二次价值"。

## 二、核心方法

### 四项关键准则

通过大量的实验分析，作者归纳出四条用于优化数据组织的准则，每一条都有其深刻的动机和理论支撑。

**G1：边界锐化（Boundary Sharpening）**

训练轨迹对优化边界处的数据分布高度敏感——尤其是训练开始和结束阶段。实验发现：
- 训练初始阶段使用**低分数据**可以稳定早期优化，让模型先掌握基础概念
- 训练结束阶段使用**高分数据**能显著提升最终性能
- 如果在开始时只用高分数据，收益几乎为零——因为固定数据量下，前期用掉高分数据意味着后期只能喂低分数据

作者提出的 SEG（Segment Ordering）算法把排序后的数据分成 L 个百分位段，每个段内随机打乱后按分数升序拼接，实现了对起止阶段的精细控制。

**G2：循环调度（Cyclic Scheduling）**

这是对课程学习"由简到难"范式的重要修正。直觉上，从简单到困难的数据安排听起来很合理，但 CL 的一个重大隐患是**灾难性遗忘**。随着模型不断接触复杂样本，它会逐渐忘记早期学到的简单知识——论文通过 PPL 分析直接证实了这一现象。

循环调度的核心思想是**周期性回顾全分数段的数据**。例如，在训练过程中不是只切换一次难度级，而是像扇面一样反复折叠回来。作者提出的 FO（Folding Ordering）算法，将排序后的数据按照步幅等分成 L 层，每层内部保持低到高的顺序。这样，即使在训练后期，模型也有机会重新消化简单的"基础知识"。

**G3：课程连续性（Curriculum Continuity）**

当循环调度在不同分数段之间切换时，存在一个隐藏问题——分数的**突变**。比如从一个高分段突然跳回低分段，会让优化器受到"冲击"，表现为梯度范数的剧烈尖峰和不稳定。

课程连续性要求分数过渡要平滑，就像锯齿波而不是方波。作者设计的 ZIG（Zig-zag Ordering）策略在 FO 的基础上，将奇数层的顺序反转，使得相邻层在端点处分数相近，形成连续的三角波形模式。

**G4：局部多样性（Local Diversity）**

即使有了全局的渐进式安排，mini-batch 内部如果样本分数过于接近，梯度多样性就会降低。模型可能过度拟合某些重复模式，丧失泛化能力。

JIT（Jittering Ordering）算法通过在局部窗口 w 内随机打乱样本，在保持整体趋势的同时注入必要的"梯度噪声"。直观来说，就是在全局规则下保留一点随机性"调味料"。

### STR 和 SAW：从准则到策略

四条准则独立验证后，作者将它们整合为两套排序策略：

**STR（Stair Ordering，阶梯排序）** 结合了 G1、G2 和 G4。它将数据集划分成若干段，稳定区域保持单调递增，而过渡区域引入 FO 折叠机制实现知识回顾。从分数分布图上看，STR 像一级级向上的楼梯。

**SAW（Saw Ordering，锯齿排序）** 在 STR 的基础上加上 G3（连续性），用 ZIG 锯齿机制替代过渡区域的 FO 折叠，保证分数过渡平滑。SAW 在分数分布图上呈现为平滑的锯齿波形。

两种策略都可以灵活选择是否应用 JIT（G4）来进一步增强局部多样性。

## 三、实验结果

论文在预训练和 SFT 两个阶段、多种数据组合上进行了详尽的实验验证。

### 预训练实验

- **模型**：Mistral 架构，规模从 160M 到 1.7B 参数
- **数据**：FineWeb-Edu（通用数据）、QuRatedPajama
- **训练量**：从 1B tokens 扩展到 50B tokens

关键数据（FineWeb-Edu，下游任务平均准确率）：
- Random（随机排序）：37.09%（基线）
- CL（课程学习）：37.61%（+0.52）
- DELT：37.35%（-0.26 vs CL）
- **STR（本文方法）**：**38.65%（+1.56）**
- **SAW（本文方法）**：**38.78%（+1.69）**

随着模型规模从 160M 增加到 1.7B，SAW 的优势持续存在且扩大，说明该方法具有良好的可扩展性。

### SFT 实验

- **模型**：Qwen3（使用官方预训练权重）
- **数据**：DeepMath-103K（数学推理）、OpenCodeInstruct（代码生成）

**数学任务（AIME24/25 平均）**：
- Random：1.30
- CL：1.78
- DELT：2.42
- STR：2.48
- SAW：**2.53**

**代码生成任务（HumanEval / MBPP 平均）**：
- Random：55.37
- CL：58.30
- DELT：59.70
- STR：60.83
- SAW：**60.48**

### Chinchilla 外推

作者还利用 Chinchilla Scaling Law 外推到大模型规模（GPT-3 175B、Llama 系列），发现**数据组织带来的收益在大规模模型上仍然保持一致**。这意味着 SAW 并不是一个只对小模型有效的小技巧，而是一个可以适配工业级训练的通用策略。

## 四、技术启示与发展方向

这项研究给 LLM 训练实践提供了几个重要启示：

**1. 让"一次计算"发挥二次价值。** 当前很多数据策展流程已经花费大量算力计算样本分数，但这些分数在完成数据选择后就被束之高阁。如果能像本文一样复用这些分数进行数据排序，几乎不增加额外成本，就可以获得显著的性能提升。

**2. 课程学习的局限性被量化揭示。** CL 的"由简到难"策略虽然直觉上成立，但在单次遍历训练中会导致灾难性遗忘。这是第一次被系统性地在 LLM 训练中量化和验证。

**3. 数据排序不是玄学，而是可设计的工程。** 四条准则的提出让数据组织从经验主义走向系统化设计——你可以根据自己的数据和模型特性，灵活组合这些准则。

**4. 平滑过渡比突变更好。** SAW 相对于 STR 的主要改进就是增加了连续性（G3），结果在大部分任务上表现更优。这提醒我们在设计训练策略时，不仅要考虑"学什么"，还要考虑"怎么过渡"。

未来的发展方向可能包括：
- 在更大的模型（如 7B-70B 规模）上进一步验证
- 探索多模态场景下的数据组织准则
- 将数据组织与动态数据选择结合起来
- 利用数据组织方法加速收敛，进一步降低训练成本

## 五、总结

这篇论文系统性地揭示了 LLM 训练中数据组织这一被低估的杠杆。四条准则——边界锐化、循环调度、课程连续性、局部多样性——构成了一个可落地的理论框架。STR 和 SAW 两种排序方法证明了这些准则的有效性，在无需额外计算开销的前提下，显著提升了模型在预训练和 SFT 阶段的表现。

当你的下一批训练数据准备就绪时，不妨多花一分钟想想：该以什么顺序喂给模型？也许答案就藏在这四条准则里。
