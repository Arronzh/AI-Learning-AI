# LLM 推理的下一个前沿：当模型学会在沉默中思考——自训练验证器与工作记忆的双重革命

📌 **核心摘要**：推理模型的自我改进在测试时（验证-精炼循环）和训练时（自训练）两个维度上都面临同一个瓶颈——验证器质量。CMU 团队提出的自训练验证（STV）通过参考溶液启发的不对称性训练验证器，在困难数学问题上将准确率翻倍，在科学推理任务上将硬题准确率从 1.5% 提升至 21%。与此同时，LSTM 之父 Sepp Hochreiter 团队提出的 RiM（Reasoning in Memory）方法，用固定记忆块替代自回归中间推理步骤，让模型学会像人类一样在"工作记忆"中内部思考。两项工作从不同角度回答了同一个问题：**LLM 如何在不依赖"大声思考"的前提下实现更高效的推理？**

---

## 一、研究背景与动机

### 1.1 测试时计算的"天花板"

自 OpenAI o1 和 DeepSeek-R1 以来，扩展测试时计算（test-time compute scaling）已成为提升 LLM 推理能力的主流范式。其核心思路很直观：给模型更多时间生成中间推理步骤，通过验证-精炼（Verification-Refinement, V-R）循环，让生成器和验证器交替工作，直至找到正确解。

这条路线已经在 IMO 和前沿数学问题上取得了突破性进展（Feng et al., 2026; Shao et al., 2025）。但隐藏在成功背后的是一组顽固的问题：

- **验证器分数膨胀**：未经训练的验证器，其打分随迭代轮次增加而单调上升，但准确率却停滞不前——模型学会了"看起来对"而非"确实对"。
- **反馈过于笼统**：当验证器说"你的解法似乎不正确"时，生成器很难据此进行有意义的修正。
- **自训练退化**：如果自训练过程中把验证器判断错误的数据也加入训练，模型反而会退化。

这些问题的根源指向同一个瓶颈——**验证器质量**。然而，"捕捉自我生成错误"这一能力本身就缺乏训练信号：如果模型自己就能发现错误，那它根本就不会犯这个错。

### 1.2 语言作为推理媒介的局限性

另一方面，现有推理方法将推理过程绑定到自回归生成——无论生成的是离散 token 还是连续表示，每一步中间计算都必须被"外部化"后才能让后续计算条件于它。

但与人类认知对比，这个范式有些奇怪。当我们解决复杂问题时，并不会把每一个中间步骤都说出来。认知心理学告诉我们，人脑有一个被称为"工作记忆"（working memory）的内部工作空间，用于保持和操作任务相关的表征，而不需要外部化中间思路。发展心理学同样观察到，虽然儿童早期依赖"出声思考"，但这种语言支架最终会被内部化。

这引出一个深层问题：**LLM 是否可以学会像人类一样，在内部进行"沉默的思考"，而不是每一次推理都依赖自回归的"自言自语"？**

两项来自不同团队的工作恰好从不同角度切入这个问题。CMU 的 Chen Henry Wu 和 Aditi Raghunathan 从验证器训练入手，提出了自训练验证（STV）框架；而 LSTM 之父 Sepp Hochreiter 团队则从推理架构入手，提出了 Reasoning in Memory（RiM）。两条路径在新的交叉点上相遇了。

## 二、核心方法

### 2.1 STV：用不对称性训练验证器

#### 关键洞察

STV 的起点是一个简单但深刻的不对称观察：**给出参考解的情况下，诊断错误的难度远低于从零开始发现错误。**

具体来说，一个模型在只有某个候选解时，可能很难判断它哪里错了——因为找出错误需要对问题空间有完整的理解。但当模型被同时展示候选解和正确答案时，任务就从"解决这个问题"变成了"比较两个解"，难度骤降。

STV 的核心策略就是利用这个不对称性：用一个"知情"的验证器（即给定参考解）作为教师，蒸馏出一个"不知情"的验证器（仅依赖候选解）。这样，在测试时运行的验证器不需要参考解，但已经学到了只有参考解才能教会的判别能力。

#### 训练流程

STV 的完整训练流程包含两个关键阶段：

**阶段一：参考条件化的教师——学生蒸馏**

- **教师函数**：V⋆(⋅|x, y_{r-1}, y⋆(x))——给定问题 x、上一轮解 y_{r-1} 和参考解 y⋆，输出判决（接受/拒绝）和自然语言反馈
- **学生函数**：Vθ(⋅|x, y_{r-1})——仅给定问题和上一轮解，输出相同的判决和反馈
- **训练目标**：使用在线策略蒸馏（On-Policy Distillation, OPD）与学生分布的 α-散度（α=0.5 即 JS 散度）匹配

为什么用 OPD 而不是简单的 SFT？作者发现 SFT 存在严重的离策略问题：学生在训练时只看到教师生成的反馈序列，但测试时一旦自己的输出偏离了教师的分布，就会遇到从未见过的前缀。OPD 则让学生在真实的 rollout 上学习。

**阶段二：判决强化学习**

在蒸馏的基础上，STV 加入了一个判决级别的 RL 组件，使用奖励 𝕀[v_r = 𝕀[y_{r-1} = y⋆(x)]] 来优化验证器的判决准确率。完整的 STV 目标函数为：

ℒ_STV(θ) = ℒ_OPD(θ) + λ · ℒ_RL(θ)

#### 验证器回环训练（ViL）

STV 的另一个关键创新是"验证器回环训练"（Verifier-in-the-Loop Training, ViL）。在训练好 STV 验证器 Vθ 后，将其冻结，然后用它来训练生成器 G 在 V-R 循环内更好地使用反馈。

每个 episode 展开一个多轮的 V-R rollout：生成器产生 y₀ → 验证器给出 (v₁, f₁) → 生成器精炼为 y₁ → ...，最终的奖励是 y_r 与参考答案的匹配度。只有生成器的参数被更新。

### 2.2 RiM：用固定记忆块实现"在沉默中思考"

#### 核心思想

RiM（Reasoning in Memory）将推理过程从自回归生成中解耦出来，思路让人联想到 LSTM 中的记忆单元——用固定 token 序列作为"记忆块（memory blocks）"，为 LLM 提供一个内部的潜在工作空间。

具体来说，每个记忆块定义为：
m_k = [<b>, <m>, <m>, ..., <m>, </b>]

其中 M 个 <m> token 形成潜在的"工作空间"，<b> 和 </b> 标记块边界。关键特点是：
- **固定而非生成**：这些 token 是固定的输入 token，不经过自回归生成过程
- **单次前向传播**：所有记忆块作为输入一部分，可以在单次前向传播中处理
- **可并行化**：不依赖序列解码，消除了自回归的顺次瓶颈

#### 两阶段课程学习

**阶段一：推理步骤监督（Reasoning Step Supervision）**

目标：教模型使用记忆块作为潜在工作空间。

将 CoT 推理轨迹分割为 T 个推理步骤，每个步骤分配一个对应的记忆块。训练模型在每一记忆块之后预测下一个推理步骤。使用自定义注意力掩码，确保每个读出只能访问之前的记忆块和输入问题，但不能访问其他推理步骤。这样模型被迫将中间计算编码到记忆块中。

**阶段二：最终答案精炼（Final Answer Refinement）**

一旦记忆块掌握了中间计算能力，阶段二移除了步骤级监督。模型改为在每个记忆块之后直接预测最终答案，并在多个记忆块上迭代精炼。这类似于垂直潜在推理中的循环模块（如 HRM, TRM），但 RiM 沿序列维度实现水平精炼。

## 三、实验结果

### 3.1 STV 的实验结果

**数学推理**

使用 Qwen3-8B 作为基座模型，在 DAPO 数学题集上评估：
- **Hard 子集**：无验证器时 pass@1 为 10.8%，STV 在 20 轮后达到 27.4%
- **Hardest 子集**（基座 pass@1=0%）：STV 达到 5.5%，而 4× 更大的 Qwen3-32B 仅 2.7%
- 增益随精炼轮次增加而持续累积，20 轮后仍未饱和

与基线对比：
- 无验证器训练：早期停滞
- 判决级 RL 验证器：仅有微弱提升
- SFT 教师跟踪：无提升（离策略问题）
- **STV：翻倍性能**

**科学推理（SciKnowEval）**

更惊人的是跨领域外推能力：
- Hardest 子集：1.5% → 21.0%（14× 提升）
- 30× 更大的 Qwen3-235B-A22B 仅 8.0%
- 4× 更大的 Qwen3-32B 仅 8.9%

**弱到强验证**

一个 4B 的 STV 验证器就能与 8B 的 STV 验证器竞争（26.4% vs 27.4%），而 1.7B 的 STV 验证器也能匹敌 8B 的未训练验证器。这表明验证器训练可以极大降低推理计算成本。

**ViL 训练效果**

最令人意外的发现来自验证器回环训练（ViL）：
- 即使从已经 RL 收敛的生成器出发，ViL 仍能带来额外的 33% pass@1 提升
- **更关键的是**，生成器第 0 轮的 pass@1（无任何验证器辅助）也提升了 30%——这意味着在 V-R 循环中训练生成器，教会了它在独立情况下就能用得上的推理能力
- 相比之下，同等计算量的标准 RL 扩展没有任何额外收益

**验证器输出分布分析**

STV 验证器与未训练验证器的关键差异：
- 未训练验证器：验证分数随轮次增加而上升，但准确率停滞——典型的奖励破解
- STV 验证器：精度-覆盖率曲线显示 3-5× 更高的精度，且精度随覆盖率增加而递增
- V-R 精炼与 Best-of-N 对比：精炼普遍优于重采样，说明验证器在"重塑（reshape）"而非"锐化（sharpen）"输出分布

### 3.2 RiM 的实验结果

使用 GSM8K-Aug 训练，在 GSM8K（分布内）和 GSM-Hard（分布外）上评估，覆盖 GPT-2 和 Llama-3.2 两个模型家族：

| 方法 | GSM8K (ID) | GSM-Hard (OOD) | 推理延迟 |
|------|-----------|----------------|---------|
| SFT w/o CoT | ~59% | ~27% | 低 |
| SFT w/ CoT | ~71% | ~38% | 高（全自回归） |
| Coconut（连续潜变量） | ~71% | ~37% | 中 |
| **RiM（Ours）** | **~72%** | **~39%** | **低（单次前向）** |

关键发现：
- **性能相当或更优**：RiM 匹配或超过 CoT 和 Coconut 的准确率
- **延迟大幅降低**：避免了自回归生成中间 token，推理效率显著提升
- **分布外泛化**：在 GSM-Hard 上的外推性能略优于基线
- **记忆块数可调**：更多记忆块持续提升性能，但收益递减

PCA 分析显示，在阶段一训练后，记忆块表征空间形成了清晰的"思维轨迹"，各记忆块间的表征差异（Δ）有序变化，而基座模型的记忆块表征几乎无差异——验证了模型确实在学习使用记忆块进行计算。

## 四、技术启示与发展方向

### 4.1 验证器训练的范式转变

STV 揭示了一个深刻的范式转变：**我们可能不应该让模型"自己改自己"（self-correction），而应该让模型"学会判断什么是对"（learn to verify）。**

这个发现的实践意义远超学术讨论。当前主流推理系统（o1, R1 等）严重依赖验证器/奖励模型的质量，而验证器的训练一直以来都是瓶颈——人工标注昂贵，自动化方法（如判决级 RL）效果有限。

STV 提出的"利用参考解提供监督信号"策略，事实上创造了一种**验证器的自举训练机制**：更好的生成器产生更有价值的训练数据 → 更好的验证器 → 更好的 V-R 循环 → 更好的生成器。只要能够定期获取新的高质量参考解，这个飞轮就可以持续运转。

### 4.2 推理效率的下一步

对于生产环境部署，STV 和 RiM 共同指向了一个重要方向：**推理效率的优化不应仅关注参数的稀疏化或量化，更应该关注推理范式的根本性改进。**

RiM 证明，模型完全可以在不生成中间 token 的情况下进行有效推理，这暗示着：
- **稀疏推理**：也许未来 LLM 不需要每次推理都走完所有层或所有 token
- **自适应计算**：困难问题多用一点"内部思考"，简单问题直接出答案
- **隐式计算**：部分推理可以发生在层内表征中，无需被外部化

### 4.3 两条路径的交叉

有趣的是，STV 和 RiM 虽然来自不同的研究动机，但在更深的层面上形成了互补：

- **STV 改进验证器**，使 V-R 循环能够持续运转，但其验证器仍然需要生成器输出外部化的解
- **RiM 改进推理架构**，使内部计算不再依赖外部化，但验证循环对它来说更难介入

一个自然的延伸是：能否结合两者——让 RiM 的记忆块表征本身成为验证器的输入？或者用 STV 训练一个能"看见"记忆块内部状态的验证器？这将是一个令人兴奋的研究方向。

## 五、总结

两项工作分别从不同角度回应了 LLM 推理的核心挑战：

**STV（Self-Trained Verification）** 证明了更好的验证器训练可以同时解锁测试时自改进和训练时自改进。其核心洞见——利用参考解与题目之间的不对称性提供训练信号——简洁而强大。14× 的科学推理准确率提升表明这个瓶颈不只是数学领域的特例。

**RiM（Reasoning in Memory）** 则证明了 LLM 可以学会使用固定记忆块作为内部工作空间进行潜在推理，无需生成中间 token。这不仅提升了计算效率，更重要的是从架构层面打破了"推理即自回归生成"的垄断。

两个方法的共同启示是：**LLM 推理的下一波进步，可能不再来自更大的模型或更多的 token 生成，而是来自更聪明的训练信号设计和更高效的推理范式。**

当模型学会在沉默中思考，我们的理解和期待也将随之重塑。
