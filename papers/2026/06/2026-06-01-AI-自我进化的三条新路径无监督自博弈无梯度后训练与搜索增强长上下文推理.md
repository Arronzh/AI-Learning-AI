# AI 自我进化的三条新路径：无监督自博弈、无梯度后训练与搜索增强长上下文推理

📌 **核心摘要**：本周 arXiv 涌现了三项从不同角度推动 AI 自我进化的代表性工作。SCOPE 让语言模型在无需人工标注、无需规则可验证答案的条件下实现自我博弈训练，在开源模型上达到甚至超越有监督 GRPO 的性能；CoRP 将权重空间中的随机扰动"折叠"回单个模型，以无梯度的方式实现后训练，用 1/10 的计算成本恢复集成推理的大部分收益；LongTraceRL 则利用搜索 Agent 的运行轨迹构建分层干扰物，并结合实体级过程奖励，系统性地改善长上下文推理。三条路径殊途同归，共同指向同一个方向：**AI 的下一次跃迁，来自于更聪明的自我改进机制，而非更庞大的计算资源。**

---

## 一、研究背景与动机

### 1.1 自我改进的"三重困境"

2024 年以来，大语言模型的自我改进（self-improvement）研究经历了爆炸式增长。从 OpenAI 的 o1 系列到 DeepSeek-R1，测试时计算扩展（test-time compute scaling）证明了"给模型更多时间思考"的价值。但在训练层面，自我改进一直面临一个"三重困境"：

- **监督信号稀缺**：RLHF 需要昂贵的人工偏好标注；GRPO/PPO 需要可验证的答案（如数学题的标准答案），对于开放式的创意写作、文档问答、推理规划等任务难以适用。
- **计算成本高昂**：无论是强化学习中的多轮采样，还是集成方法中的多次前向传播，额外的推理开销都在不断累积。
- **反馈信号粗糙**：最终答案是否正确这种二值信号，无法指导模型在中间的推理步骤上做出改进。

本周 arXiv 上的 SCOPE、CoRP 和 LongTraceRL 分别从三个维度给出了不同的破局思路。

### 1.2 从"被动学习"到"主动进化"

一个更深的背景在于：当前主流训练范式本质上仍然是"被动学习"——模型依赖人类提供的数据分布进行模仿或对齐。真正的"主动学习"需要模型能够自我生成训练信号、自我验证、自我修正。这三篇工作共同展现了一个趋势：**AI 正在从需要人类提供训练数据的阶段，迈向能够自主构建训练循环的阶段。**

## 二、核心方法

### 2.1 SCOPE：让模型学会"出题"和"解题"的共同进化

**核心洞察**

SCOPE（Self-Play via Co-Evolving Policies for Open-Ended Tasks）的核心想法简洁而优雅：如果让模型同时扮演"出题老师"和"解题学生"，那么这两者可以相互促进，形成一个无需外部监督的训练循环。

具体来说，SCOPE 在训练过程中维护两个策略：

- **Challenger（挑战者）**：从一个文档中提取关键信息，生成基于该文档的开放性问题。这些问题需要足够有挑战性——太简单则 Solver 学不到东西，太难则 Solver 答不出来。
- **Solver（解答者）**：针对 Challenger 提出的问题，通过多轮检索来寻找答案并组织成完整的回复。

**自裁判机制**

SCOPE 的高明之处在于第三个角色——Self-Judge。训练开始时的原始模型被冻结，作为"裁判"，它从源文档中写出一份针对该问题的评分细则（rubric），然后根据这份细则给 Solver 的回答打分。这个设计避免了"用同一个模型既当选手又当裁判"的自我褒奖问题。

**共同进化动力学**

SCOPE 最关键的设计是"共同进化"——Challenger 和 Solver 交替更新。如果只更新 Solver 而不更新 Challenger，Solver 很快就能答对所有问题，训练信号消失。但如果 Challenger 也一起进化，它会自动调整问题的难度，始终保持在 Solver 能力边界的"最近发展区"。这种动力学与 GAN（生成对抗网络）有异曲同工之妙，但训练目标不是"欺骗"而是"促进学习"。

### 2.2 CoRP：把扰动"拧"成一个模型

**从 RandOpt 到 CoRP**

RandOpt 提出了一种有趣的思路：不再用梯度下降更新模型参数，而是在预训练模型周围采样多个高斯扰动，然后从中选出 Rewarded 最高的 Top-K 个模型，在推理时做集成。这种方法在匹配计算预算的前提下，取得了与 PPO/GRPO 相当的性能。

但 RandOpt 有两个实际问题：一是推理时要做 K 次前向传播，延迟和成本线性增长；二是对于自由文本生成任务（如创意写作），集成策略难以优雅地融合多个输出。

CoRP（Consolidating Rewarded Perturbations）的出发点是：**能否把这 K 个扰动直接融合进一个模型，让集成发生在参数空间而非输出空间？**

**低秩结构的发现**

作者在 25 组模型-任务对上做了分半分析（split-half analysis），发现了一个令人惊讶的规律：被奖励选择的扰动模型之间的差异，呈现出可重复的低秩结构。换句话说，那些"效果好"的扰动并非随机分布在权重空间中，而是集中在某个低维子空间里。

**CoRP 的三步操作**

基于这个几何发现，CoRP 设计了三个操作：

1. **Reward-Weighted Aggregation**：对 Top-K 扰动按奖励加权平均，先得到一个粗粒度聚合。
2. **Compatibility-Aware Reweighting**：考虑到不同扰动之间的兼容性差异（有些扰动在一起会互相抵消），加入了一个重加权步骤。
3. **Held-Out Validation Gate**：在验证集上做一次门控筛选，确保输出的参数确实优于基座模型。

整个过程中**没有任何梯度流经语言模型**——所有操作都在参数空间中进行。最终输出一个单独的、可直接部署的模型。

### 2.3 LongTraceRL：用搜索 Agent 的轨迹教模型长上下文推理

**长上下文推理的困境**

长上下文推理（long-context reasoning）的难点不仅在于"长"，更在于"干扰"。当给定一篇包含大量无关信息的文档时，LLM 常常迷失在文本森林中，把关键信息淹没在注意力池的深处。

现有的 RLVR（Reinforcement Learning with Verifiable Rewards）方法有两个问题：

- **干扰物太简单**：随机采样的干扰文档往往与问题明显无关，模型很容易过滤掉它们。
- **奖励信号太稀疏**：只有最终答案的正确/错误，无法监督中间推理过程。

**搜索 Agent 轨迹带来的灵感**

LongTraceRL 的切入点让人眼前一亮：既然搜索 Agent 在执行多步检索时天然会产生"读了但没用"和"搜到了但没打开"两种类型的干扰文档，那为什么不直接用这些轨迹来构造训练数据呢？

具体来说，作者用知识图谱随机游走生成多跳问题，然后让一个搜索 Agent 去回答这些问题，记录下它的完整运行轨迹。这样就得到了三种文档：

- **相关文档**：Agent 最终引用的文档
- **高混淆度干扰**：Agent 读了但没有引用的文档——它们看起来相关，但实际不包含关键信息
- **低混淆度干扰**：Agent 搜到了但根本没打开的文档——明显不相关但增加长度

**Rubric Reward：实体级过程监督**

比数据构造更有价值的是奖励设计。作者提出了 Rubric Reward：每个多跳问题都对应一条推理链（从起始实体到目标实体的路径），路径上的每个实体都是一个"检查点"。如果模型的最终答案正确，那就进一步检查它在回答中是否提到了路径上的关键实体，并根据覆盖的实体数量给予细粒度的奖励。

这个设计只在答案正确的样本上应用奖励（positive-only strategy），有效防止了奖励破解（reward hacking）。

## 三、实验结果

### 3.1 SCOPE

SCOPE 在 Qwen2.5-7B、Qwen3-8B 和 OLMo-3-7B 三个模型上进行了验证：

- **开放任务性能**：在 8 个基准测试上，最高提升达 +10.4 个百分点
- **与 GRPO_data 对比**：匹配甚至超越了基于约 9K 精心策划提示的 GRPO 训练
- **跨任务迁移**：虽然只针对开放任务训练，但在 7 个保留的短式 QA 基准上也取得了最高 +13.8 个百分点的提升，**全面超越 GRPO_data**（在所有三个模型上均实现）

消融实验揭示了几个关键发现：

- **共同进化是必须的**：固定 Challenger 后训练 Solver，性能提升有限；只有两者共同进化才能持续改进
- **瓶颈在裁判**：Rubric 生成质量是自裁判流程中的最大短板——如果裁判写不好评分标准，后续的训练信号就会偏离方向
- **检索和合成各有贡献**：不同任务对检索能力和合成能力的依赖程度不同，两者相互补充

### 3.2 CoRP

CoRP 在从 0.5B 到 8B 的五个模型、涵盖数学、代码和创意写作的五项任务上评估：

- **平均提升 8.1 个百分点**：相比基座模型
- **超越 RandOpt 单次推理 6.5 个百分点**：仅使用 RandOpt 十分之一的扰动预算
- **恢复 50 次投票集成的一半收益**：但只需要一次前向传播

这意味着 CoRP 找到了参数空间中的"高效通路"——大部分有用信息集中在低维子空间中，可以通过简单的加权运算提取出来。

### 3.3 LongTraceRL

在 4B 到 30B 的三个推理模型上，通过了五个长上下文基准测试：

- **全面超越强基线**：包括直接 RLVR 训练、上下文学习等
- **实体级过程奖励有效**：相比纯结果奖励，Rubric Reward 显著提升了中间推理步骤的质量
- **搜索轨迹数据的优势**：基于 Agent 轨迹构造的层级干扰物，比随机采样或单次搜索构造的数据更具挑战性，训练出的模型在真实长上下文场景下表现更好

## 四、技术启示与发展方向

### 4.1 自我改进范式的"三足鼎立"

这三篇工作从不同角度回答了同一个核心问题：**当人类监督信号越来越昂贵时，模型如何自我驱动进化？**

- **SCOPE 回答了"无监督训练"**：通过双角色共同进化 + 自裁判，在无需规则可验证答案的开放任务上实现了自我博弈
- **CoRP 回答了"无梯度训练"**：在参数空间中寻找高效通路，把多次推理的收益折叠进一次前向传播
- **LongTraceRL 回答了"精细反馈"**：利用已有的 Agent 行为轨迹提供结构化训练数据，用实体级奖励替代二值奖励

### 4.2 Co-Evolution 的通用性

SCOPE 的"共同进化"框架值得特别关注。它的设计不限于特定任务类型——Challenger 和 Solver 的角色定义足够灵活，可以适配到各种场景。这让人联想到 AlphaGo 中自我对弈的成功：当 AI 能够同时扮演挑战者和回应者时，它就不再受限于人类数据的边界。

但共同进化也有其隐忧。如果裁判的质量无法保证，整个飞轮就会跑偏。SCOPE 的消融实验明确指出 Rubric 生成是瓶颈，这意味着如何在无监督条件下训练出更好的裁判模型，是下一步的关键。

### 4.3 参数空间操作的潜力

CoRP 的结果暗示了一个更大的可能性：**语言模型的后训练不需要通过梯度下降来完成。**

传统观念认为，微调必须通过反向传播更新参数。但 CoRP 表明，只要理解了优胜参数在权重空间中的分布结构，简单的向量运算就能达到类似的效果。这为模型合并、联邦学习、持续学习等领域开辟了新思路。

特别是，"分半分析揭示低秩结构"这个发现本身，可能就是比 CoRP 更有价值的贡献——它为理解语言模型的损失景观（loss landscape）提供了新的测量工具。

### 4.4 搜索 Agent 作为数据生成器

LongTraceRL 的另一个启发是：**不要低估中间产物（byproduct）的价值。** 搜索 Agent 在运行过程中产生的"读了但没用"的文档——这种日常工作中的"垃圾信息"——恰恰是训练长上下文推理的高质量干扰数据。

这暗示了一个更大的方向：随着 Agent 系统在生产环境中广泛部署，它们运行过程中产生的日志、轨迹和决策路径，本身就可以成为一种持续的训练数据来源。这可能是真正的"数据飞轮"——AI 系统不是退出生产环境去收集数据，而是在运行时自然生成训练信号。

## 五、总结

| 工作 | 核心思想 | 关键结果 | 最大亮点 |
|------|---------|---------|---------|
| SCOPE | 双策略共同进化 + 自裁判 | 开源模型上超越有监督 GRPO | 无监督开放任务自我博弈 |
| CoRP | 扰动折叠 + 参数空间融合 | 1/10 扰动预算超越 RandOpt | 训练和推理都不需要梯度 |
| LongTraceRL | 搜索轨迹构造干扰 + 实体级 Rubric 奖励 | 全面超越长上下文基线 | 利用 Agent 中间产物作为数据 |

三篇论文的共同启示：**AI 自我改进的下一波浪潮，不再取决于我们能否收集更多人类数据或调度更多算力，而取决于我们能否设计出更聪明的自我学习机制。** 当模型学会出题、学会在参数空间中自我调整、学会从自己的行为轨迹中学习，人类需要做的事情就越来越少了——而这，也许正是我们通向通用人工智能最务实的道路。
