# SearchSwarm：当 LLM 学会"分配任务"，轻量模型如何击败十亿参数巨无霸？

📌 **核心摘要**

> 上下文窗口不够用怎么办？清华、北大、蚂蚁集团和中国人民大学组成的研究团队提出了一个全新的思路：**让大模型学会"分配任务"**——主 Agent 将复杂任务拆解成子任务，分派给子 Agent 执行，子 Agent 只返回压缩后的结果。这种被称为"委派智能"（Delegation Intelligence）的能力，使得仅 30B-A3B 参数的 SearchSwarm 模型在 BrowseComp 上达到 68.1 分，超越了 GPT-5.2-Thinking（65.8）和 DeepSeek V3.2（67.6），甚至能与 10 倍以上参数的模型同台竞技。

---

## 一、研究背景与动机

### 1.1 大模型的长程任务困境

随着 LLM 被广泛应用于复杂的真实世界任务，一个根本性矛盾日益凸显：**任务复杂度可以无限增长，但模型的上下文窗口始终是有限的。**

现有的解决方案大多是"被动式"的：
- 超过长度阈值后**截断或丢弃历史信息**
- 只保留最近几轮工具调用结果
- 对轨迹进行自动摘要压缩

这些方法的问题在于：它们缺乏预先规划，总是在上下文预算耗尽后才采取行动，或者按照固定规则随意丢弃信息。

### 1.2 "委派范式"：一种更智能的上下文管理

研究团队提出了一种新的范式：**主 Agent 将任务分解后分派给子 Agent 执行**，子 Agent 在独立的上下文中运行，只将压缩后的结果返回给主 Agent。

这种方法的优点在于：
- 子 Agent 的探索成本被隔离在自己的上下文中
- 主 Agent 的注意力始终集中在高层协调上
- 返回的摘要天然是一种"内容感知的压缩"

> "表面上这是一个多 Agent 系统，但实际上它是**单 Agent 的上下文管理**——只是模型用自己生成的摘要替换了固定规则的截断。"

## 二、核心方法

### 2.1 SearchSwarm 架构

SearchSwarm 采用 **"主分配-子执行"（Main-Distributes, Sub-Executes）范式**，整体流程如下：

1. **主 Agent** 接收用户查询，规划整体研究方案
2. 主 Agent 将复杂问题拆解为多个子任务
3. 通过 `call_sub_agent` 工具将子任务分派给子 Agent
4. **子 Agent** 在独立上下文中执行搜索和验证
5. 子 Agent 返回带引用的压缩报告
6. 主 Agent 汇总各子报告，形成最终答案

### 2.2 四大设计原则

研究团队为 Harness 设计了四条关键原则：

**① 鼓励委派（Encouraging Delegation）**
- 多步信息收集是 token 密集但认知浅的工作
- 主 Agent 应将这些工作交给子 Agent
- 只有当子任务足够浅时，主 Agent 才自行执行

**② 全面简报（Comprehensive Briefing）**
- 子 Agent 从零开始，没有预先的研究背景
- 简报不仅是任务描述，还应包括：
  - 这个子任务为什么重要
  - 已经确定了什么
  - 还有什么不确定
  - 哪些方向已经尝试过或排除了
- 这好比是向新加入的合作者介绍项目进展

**③ 主 Agent 保留核心判断（Core Judgment）**
- 只有主 Agent 拥有全局视图
- 子 Agent 负责收集证据和验证假设
- 所有方向性决策由主 Agent 独立做出：
  - 选择哪个假设继续深挖
  - 何时终止调查
  - 如何裁决冲突的报告

**④ 引用溯源报告（Citation-Grounded Reporting）**
- 子 Agent 的每个重要结论必须附带内联引用，指向具体来源 URL
- 主 Agent 可以验证结论的可靠性
- 最终答案同样包含内联引用，实现端到端的可追溯性

### 2.3 数据合成与训练

研究团队面临的核心挑战是：**委派智能的训练数据在自然文本中极为稀缺**——自然语料很少包含多 Agent 协作行为。

解决方案分为三步：

**Step 1：用 Harness 生成高质量轨迹**
- 在 Harness 引导下，让模型在深度研究任务上执行委派行为
- 记录完整的执行轨迹（思考→工具调用→环境返回）

**Step 2：筛选与过滤**
- 只保留主 Agent 轨迹中答案正确的样本
- 子 Agent 轨迹仅在对应主轨迹正确时保留
- 去除不良行为模式（重复工具调用、幻觉引用、工具误用）

**Step 3：监督微调**
- 采用下一个 token 预测 + 环境掩码的训练目标
- 损失只计算模型输出部分，环境返回全部掩码
- 这样训练模型学会给定上下文产生正确的推理和工具调用

### 2.4 委派即上下文管理（重要洞察）

论文的一个重要洞察是：**委派机制本质上是上下文管理的一种形式。**

当主 Agent 调用 `call_sub_agent` 时，模型在下一个推理步骤中只基于简报进行推理，而不是基于完整的历史。子任务执行完成后，只有摘要报告重新进入主上下文。

与传统方法相比（丢弃历史、自动摘要是固定规则决定的），这里**简报和报告都由模型自己生成**——这是一种更智能的、内容感知的压缩策略。

## 三、实验结果

### 3.1 全面领先的表现

| 基准测试 | SearchSwarm（30B-A3B） | 上一代最佳（同规模） | GPT-5.2-Thinking | DeepSeek V3.2（671B） |
|---------|----------------------|-------------------|-----------------|---------------------|
| BrowseComp | **68.1** | 67.9（MiroThinker-1.7-mini） | 65.8 | 67.6 |
| BrowseComp-ZH | **73.3** | 72.3（MiroThinker-1.7-mini） | — | — |
| GAIA | **82.5** | 80.3（MiroThinker-1.7-mini） | — | 76.4（GPT-5） |
| xbench-DeepSearch | **80.8** | 78.0（LongSeeker） | — | — |

关键发现：
- 在所有四个基准测试中，SearchSwarm 实现了 **30B-A3B 规模的最佳表现**
- 相比基础模型（无上下文管理，43.4），训练带来了 **24.7 个百分点的绝对提升**
- 在 GAIA 上甚至超过了 GPT-5（76.4）和 Seed-2.0-Pro（78.6）

### 3.2 Harness 有效性验证

在 BrowseComp 的 200 题子集上：
- 基础框架：47.7
- +`call_sub_agent` 工具（仅参数描述）：50.0（+2.3）
- **完整 Harness：57.7（+10.0）**

仅仅提供委派工具只有小幅提升，而完整的 Harness 设计带来了大幅提升。

### 3.3 跨基座模型泛化

使用完全相同的训练数据微调 **Qwen3-30B-A3B-Thinking-2507**，同样获得显著提升：
- BrowseComp 200 题子集：66.5
- BrowseComp-ZH：64.0
- 超越了 RedSearcher 和 LongSeeker 在各自最佳配置下的结果

这表明训练数据本身就具有高质量，不依赖于特定基座模型。

### 3.4 泛化到单 Agent 场景

将 SearchSwarm 的 `call_sub_agent` 工具禁用，回归纯单 Agent：
- SearchSwarm：52.0（BrowseComp 子集）
- Tongyi DeepResearch（原始）：43.5
- 提升 **8.5 个百分点**

训练数据中完全不包含单 Agent 轨迹，但委派智能训练带来的**系统化问题分解、子问题有序求解、整体研究进度的维护能力**，自然地泛化到了单 Agent 场景。

### 3.5 泛化到开放式深度研究

在需要长文本合成回答的基准测试上：
- SearchSwarm 平均 64.2 vs 基础模型 50.0（提升 14.2 分）
- ScholarQA-v2 提升最为显著（+32.7 分）
- 接近 OpenAI DeepResearch（64.9）

> 训练数据全部来自短答案任务，**零样本泛化到开放式长文本场景**，这才是委派智能训练真正的威力所在。

## 四、技术启示与发展方向

### 4.1 委派智能：AI Agent 的下一个重要能力

这项研究揭示了 AI Agent 发展的一个关键方向：**与其不断扩展模型规模或上下文窗口，不如教会模型更智能地管理有限的资源。**

委派机制本质上是一种"元认知"能力——模型不仅要会做事，还要知道**什么事该自己做，什么事该交给别人做**。

### 4.2 开源生态的重大贡献

研究团队承诺开源：
- Harness 完整代码
- 模型权重（SearchSwarm-30B-A3B）
- 训练数据

这将极大推动委派智能和 Multi-Agent 协调方向的研究进展。

### 4.3 现有局限

- **多轮提升不显著**：研究团队在另一篇论文中指出，即使有精准的指导信号，可靠的多轮改进仍然是当前 DRA 架构无法实现的
- **单层委派**：目前限制为一层（子 Agent 不能再委派），限制了更复杂的任务分解能力
- **大规模验证**：目前集中在 30B-A3B 规模，更大规模的委派智能训练效果有待验证
- **模型规模限制**：虽然与更大模型竞争，但绝对性能仍有差距

---

## 五、总结

SearchSwarm 是委派智能领域的开创性工作，证明了通过精心设计的 Harness 引导和 SFT 训练，**轻量级模型可以在长程研究任务上达到甚至超越超大规模模型**。

核心贡献可概括为：
1. 首次系统探索了**委派智能**在超长程 Agent 任务中的应用
2. 设计了包含四大原则的 **Harness**，有效引导模型产生高质量的委派行为
3. 利用 Harness 轨迹合成了高质量的 SFT 数据，将委派行为内化到模型权重
4. 训练出的 SearchSwarm 在多个基准测试上达到同规模最佳，展现跨场景泛化能力

对 AI Agent 开发者而言，这一思路具有重要的实践意义：**与其一味追求更大的模型和更长的上下文，不如让模型学会"分配任务"**——正如一位优秀的项目经理，不一定自己什么都会，但知道该怎么把工作分配出去。
