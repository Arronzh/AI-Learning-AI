## 一、研究背景与动机

大模型正从"能跑"迈向"好用"——但这一过程充满了折中与瓶颈。

**部署阶段的适应性问题**：当下大多数 prompt learning 方法（GEPA、ACE 等）假设任务分布是静态的，但真实世界的 Agent 面对的输入流是跨领域、跨格式的混合流。单一 prompt 难以容纳多样任务，容易产生**跨数据集干扰**。

**推理模型的可控性问题**：DeepSeek-R1 等推理模型的 Chain-of-Thought 极长，且内部行为分布高度不确定。现有的激活操控方法（activation steering）依赖检测已生成文本中的行为特征，但无法有效预测未来行为，且易导致输出质量退化。

**推理效率瓶颈**：长 CoT 导致 KV Cache 成为显存瓶颈，现有压缩方法大多假设各层、各头的预算分配是均匀的。而实验表明，推理模型的 KV 需求在实际中呈现高度非均匀的"**推理波**"模式。

今日三篇工作分别从**适应**、**控制**、**压缩**三个维度切入，给出了各自的优雅解。

## 二、核心方法

### 🧩 EEVEE：路由-提示协同演化的多数据集 Prompt Learning

**机构**：上海交通大学 x 普林斯顿大学

**核心思路**：与其让模型用一个 prompt 应对所有任务，不如维护一个**提示集** + 一个**路由器**，让路由器为新输入分配最合适的提示。

**关键创新——路由-提示协同演化**：

1. **初始化阶段**：在多领域混合数据上独立学习一组多样化的 prompt，构建 Pareto 前沿池，通过贪心覆盖选择 K 个差异化提示。
2. **探索阶段**：交替优化路由器与提示集——先固定提示集，用突变+反思搜索更优路由策略；再固定路由器，为每个路由槽位独立进化提示。
3. **收敛阶段**：路由策略稳定后，锁定路由器，投入更大预算在每个槽位上精细优化提示。

路由器的评分函数融合了三个维度：下游准确率（A）、分簇内聚性/分离性（C）、负载均衡（B），并设计了退火权重机制。

### 🎯 FPCG：基于未来行为预测的推理模型操控

**机构**：Fraunhofer HHI · 东北大学 · KAIST

**核心发现**：推理模型的内部表征中存在**两类截然不同的特征**——

- **检测特征（Detection Features）**：表征已生成文本中是否存在某种行为。这是现有 steering 方法依赖的对象。
- **预测特征（Prediction Features）**：表征模型在未来的生成中是否会表现出某种行为，反映了模型的"意图"。

实验表明，检测特征对未来行为的预测能力很差，而预测特征可用线性探针以 **64%~91% 的准确率**预判行为概率。

**FPCG 算法**受此启发：逐句生成时，为每个推理步生成多个候选句子，用 MLP 探针打分，选择最能提升/降低目标行为概率的句子。完全避免了激活操控导致的输出质量退化。

### 📊 ReasonAlloc：推理模型的分层 KV Cache 预算分配

**机构**：清华大学 · 香港城市大学 · 北京大学 · 中科院自动化所

**核心发现**：经分析，推理模型的 KV 需求在层间呈现非单调的"**推理波**"模式：

- **浅层**：需求高，承载全局语义感知
- **中层**：需求低且振荡，局部逻辑推理
- **深层**：需求突增，输出前的整体验证

且该模式在不同架构间差异显著（跨架构方差是跨任务方差的 **3.8 倍**），但同一模型内高度稳定。

**ReasonAlloc 框架**：

1. **离线层间预分配**：用少量随机 prompt 校准每层的需求曲线，经幂次平滑 + 边界保护生成稳定预算。
2. **在线头间动态路由**：每 Δ 个 token 重新评估各注意力头的效用，将预算动态路由到当前信息密集的头，并用平滑算子防止"饿死"循环。

与 R-KV 等 token 评分方法正交，可作为即插即用模块。

## 三、实验结果

### EEVEE 关键结果

| 设置 | 指标 |
|------|------|
| Qwen3-4B-Instruct 四基准平均分 | **51.75**（对比基线 +10.38，超 GEPA 14.02，超 ACE 16.83） |
| DeepSeek-V3.2 四基准平均分 | **64.07**（对比基线 +24.32，超 GEPA 8.24） |
| 累积任务保持（四任务后） | +41.53（GEPA 为 -15.36，ACE 为 -18.58） |
| Token 开销 | 4.32k / 例，仅 ACE 的 **1/5** |
| 跨模型泛化 | Qwen 学到的 prompt 直接用于 DeepSeek-V3.2 仍有 +14.35 提升 |

### FPCG 关键结果

- 预测特征在 20/24 测试设置中二值准确率超过 **70%**，拒绝行为预测高达 **91%**
- FPCG 在操控强度上与激活操控相当，但**输出质量几乎无退化**（过滤率接近 0%）
- 在 gpt-oss-20b 上，FPCG 在激活操控完全无效的两个数据集中实现了成功操控

### ReasonAlloc 关键结果

| 预算 | R-KV | ReasonAlloc | 提升 |
|------|------|-------------|------|
| MATH-500 @ 128 token | 52.77% | **57.80%** | +5.03 |
| MATH-500 @ 512 token | 76.48% | **82.50%** | +6.02 |
| AIME 2024 @ 256 token | 10.42% | **20.00%** | +9.58 |
| AIME 2024 @ 1024 token | 45.42% | **49.17%** | +3.75 |

消融实验表明，层间分配 + 头间路由两者缺一不可，完整 ReasonAlloc 在所有预算设置下均优于单一组件。

## 四、技术启示与发展方向

### 1. 从 "一个 prompt 走天下" 到 "路由 + 提示集"

EEVEE 的核心理念——用路由器解耦任务间的干扰——是对当前 Agent 架构的一个深刻修正。未来方向包括：
- 扩展到更细粒度的任务发现（无预设类别数）
- 在完全在线场景下实现全反射式学习（无需 ground-truth）
- 探索路由器本身的稀疏化与小型化

### 2. 推理模型的可控性：从 "事后检测" 到 "事前预测"

FPCG 从根本上区分了"检测"与"预测"两种内部特征，开启了推理模型操控的新范式：
- 可应用于安全对齐：在推理过程中识别并避免有害行为
- 可扩展到更复杂的行为空间（非二值场景）
- 与 SAE 等可解释性工具的结合将释放更大潜力

### 3. KV Cache 压缩：非均匀分配是必选项

ReasonAlloc 用实证推翻了"均匀分配 + 单调递减"的假设：
- 推理模型的 KV 需求是架构依赖的、非线性的
- 头间动态路由在高压缩比下至关重要
- 对 MoE 模型、多模态推理模型的适配是自然延伸

## 五、总结

今日的三篇论文从三个不同角度回应了当前 AI 系统面临的核心矛盾——**模型越强大、越智能，其部署和调控的复杂度也越高**。

- **EEVEE** 告诉我们：适应多领域任务需要专门化，而非一个 prompt 统吃。
- **FPCG** 告诉我们：理解模型"在想什么"比"说了什么"更有利于控制它。
- **ReasonAlloc** 告诉我们：KVCache 不是均匀的水桶，应该按需分配。

这三项工作在方法论上都采用了"先分析、后设计"的路径——通过对模型内部机制的深入理解来驱动技术方案创新，而非靠直觉堆叠技巧。这种研究范式值得我们持续关注。
