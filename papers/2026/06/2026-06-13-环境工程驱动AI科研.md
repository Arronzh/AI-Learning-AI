# 别再给AI写剧本了：清华与智谱AI提出"环境工程"新范式，11美元跑出顶级科研成绩

📌 **核心摘要**

> 当大多数研究还在为AI Agent设计精密的"工作流剧本"时，清华大学与智谱AI联合团队另辟蹊径——他们提出了**EurekAgent**，一个以"环境工程"而非"流程控制"为核心的自主科研系统。该系统在数学发现、内核工程和机器学习三大领域的多项任务上刷新了SOTA，其中26圆 packing 问题的求解仅花费不到11美元API费用。同期，香港中文大学发布的**FORGE基准**则揭示了一个令人不安的事实：联网搜索增强的LLM在面对单页被污染的网页内容时，有高达27%的概率推荐虚假商品——推理能力越强，反而越容易被欺骗。这两篇论文共同指向一个趋势：AI系统的可靠性，正从模型能力的比拼，转向**环境设计**的较量。

---

## 一、研究背景与动机

### 自主科研智能体的"新瓶颈"

大语言模型的飞速发展，让AI自主科研从愿景走向现实。从FunSearch到AIDE，从AlphaEvolve到The AI Scientist，一系列系统已经证明：给定一个可优化的目标和执行环境，LLM Agent能够自主提出假设、编写代码、运行实验并迭代方案。

然而，一个耐人寻味的发现打破了行业共识：在ResearchClawBench基准上，**通用型CLI Agent（如Claude Code和Codex）的表现，竟然超越了所有专为科研设计的专用Agent系统**。

这意味着什么？清华与智谱AI团队的答案很明确：**随着基础模型能力的持续提升，自主科研的瓶颈正在从"如何设计Agent的工作流"转向"如何设计Agent的环境"**——这个环境包括了资源、约束和接口，它们共同塑造了Agent的行为模式。

这就好比一位优秀的博士生：他的生产力不是来自于导师每分钟的指令，而是来自于**责任机制、研究自主权、准确反馈、同行协作和导师监督**。EurekAgent正是这一理念的系统实现。

### 搜索增强LLM的"推荐陷阱"

与此同时，香港中文大学团队关注了一个与日俱增的现实威胁：越来越多的AI助手通过联网搜索为用户提供消费推荐，但搜索引擎可能被"生成式引擎优化（GEO）"技术所污染。2026年3月15日，央视3·15晚会曝光了一个地下产业——商业GEO运营者通过在网络上植入虚假评论，能在数小时内让一个虚假品牌出现在主流AI助手的推荐结果中。

**问题不再是"模型有多聪明"，而是"环境是否可信"**。

## 二、核心方法

### EurekAgent：环境工程的四个维度

EurekAgent的核心理念非常简洁：使用现成的通用CLI Agent作为基本节点，围绕Agent设计一个精心编排的外部环境，而非试图控制Agent内部的每一步行为。

其运行流程是一个简单的三轮循环：

```
准备 → [提案 → 并行的 {实施_1, 实施_2, ..., 实施_P}] × R轮
```

在这一循环中，环境只负责外层协调：初始化工作空间、切换阶段、设定目标和交付物、暴露工具和资源接口、记录排序方案、以及执行预算和时限约束。**Agent在这些边界内完全自由地决定自己的研究策略**。

更关键的是EurekAgent围绕**四个环境工程维度**的设计：

**① 权限工程**
系统通过Docker容器实现运行级隔离，被评估的解决方案和测试数据位于Agent可见的工作区之外，仅通过安全评分服务暴露接口。Agent可以提交候选方案并获得官方评分，但**无法查看或篡改评估器本身**。同时，同轮的并行实施会话之间相互隔离，防止过早收敛到单一方向。

**② 制品工程**
每一次运行都有完整的文件系统和Git版本控制。制品的结构遵循严格的模板（问题描述→配置→签到记录→排名方案），构建起一份"共享进度记忆"。这使得Agent可以追踪自己的历史方案、关联日志与结果，而人类研究者也可以通过Web监控界面一览所有会话的完整轨迹。

**③ 预算工程**
Agent的探索受到时间和API成本的双重约束。预算不仅是一个截止条件，更是一种**引导工具**：预算的多寡直接影响了Agent选择探索策略还是利用已知方向。实证表明，合理设置预算可以显著提升Agent的探索效率。

**④ 人机协同工程**
研究者可通过Web监控界面实时观察评分曲线，在必要时暂停运行、调整参数或介入干预。Agent也被设定为在检测到配置含糊或执行环境异常时主动请求人类说明，而非盲目继续。

### FORGE基准：可复现的内容污染评估

FORGE的核心设计简洁而严谨。它模拟了真实世界中搜索增强LLM的工作流程：

```
用户查询 → 实时网络搜索 → 前K个搜索结果 → LLM阅读理解 → 推荐输出
```

但为了避免实际污染网络，FORGE采用**局部替换**的方式：在冻结的搜索结果中，将选中文档里的真实品牌名称重写为虚构的"假品牌-产品"组合，同时保留文档的排名、URL、来源归属、长度和周围上下文。这样一来，LLM推荐结果的变化只能来自品牌替换本身。

FORGE涵盖了**15个产品类别、225个真实产品、5种消费场景**，评估了12个商用和开源LLM。

## 三、实验结果

### EurekAgent：低成本的SOTA突破

在数学发现、内核工程和机器学习三大领域的5项任务中，EurekAgent全面刷新SOTA：

| 任务 | 此前最佳（人类） | 此前最佳（AI） | **EurekAgent** |
|------|:---------:|:---------:|:---------:|
| 26圆 Packing（↑） | ~2.634 | 2.635986 | **2.635999** |
| Erdős最小重叠问题（↓） | 0.380927 | 0.380876 | **0.380870** |
| 一阶自相关不等式（↓） | 1.509730 | 1.502863 | **1.502861** |
| TriMul核函数（↓） | 2096.04μs | 2247.78μs | **2005.03μs** |
| MLE-Bench子集（↑） | N/A | 71.43% | **85.71%** |

其中，以Claude Code为CLI Agent、GLM-5.1为基础模型的配置下，三项数学任务的平均API成本不到**17美元**，26圆 packing任务的成本更是低至11美元——这比绝大多数研究者手动实验的成本还要低得多。

### FORGE：没有模型能免疫

FORGE的实验结果令人警醒：

- **普遍脆弱性**：所有12个模型在top-3替换攻击下，被愚弄率从13.3%到73.8%不等。即使**单页污染**，也有高达27%的被骗概率。
- **规模与安全性无关**：更大的模型并不更安全——Gemini 3.1 Pro被愚弄的次数竟是Gemini 3 Flash的约3倍。
- **推理反成弱点**：启用推理的模型比禁用的版本更易受骗（差距最大达18个百分点）。更令人担忧的是，**模型在推荐虚假产品时会编造"社会证据"**——虚假社交证明标识的出现频率是正常推荐时的1.5~11倍。
- **现有防御失效**：怀疑提示（skepticism prompting）、模型先验共识和交叉文档一致性过滤三种防御措施均未能可靠缓解攻击。怀疑提示甚至在某些模型上适得其反，使攻击成功率增加多达44个百分点。

**最脆弱的类别恰恰是用户最需要推荐的领域**：餐饮（平均81.7%被愚弄率）、个人服务（60.6%）和保健品（60.0%）。

## 四、技术启示与发展方向

两篇论文共同揭示了一个正在形成的共识：**AI能力的下一步跃升，很可能不是来自更大的模型或更巧妙的算法，而是来自更聪明的环境设计。**

EurekAgent告诉我们：当基础 Agent 足够强大时，"保姆式"的工作流控制反而可能限制其潜力。更聪明的做法是像设计一个实验室一样设计Agent的运行环境——设定清晰的边界、提供准确的反馈、赋予充分的自主权，然后放手让Agent自己去探索。

而FORGE则从反面验证了这一观点：同样是因为环境（搜索结果的污染）而非模型本身的缺陷，才导致了推荐系统失效。**当环境不设防时，最强的推理能力也可能变成最强的"被骗能力"**。

两个值得关注的发展方向：

1. **环境工程作为AI系统设计的一等公民**：正如软件工程从编写代码发展到设计架构和基础设施，AI Agent的设计也需要从写Prompt的层面提升到设计运行环境的层面——权限、制品、预算、人机协同，应当成为每个AI系统的标准模块。

2. **"抗污染"环境协议**：FORGE团队提出的防御方案大多失效，说明需要更根本性的方案——比如建立内容可信度评分体系、跨来源交叉验证的标准协议、以及"可追溯的数字身份"机制，让模型能够判断信息来源的可信度。

## 五、总结

清华大学与智谱AI的EurekAgent，以环境工程为核心理念，用不到11美元的API成本就刷新了数学发现的SOTA，为自主科研AI的发展提出了一个极具启发性的新范式。而香港中文大学的FORGE基准，则冷静地揭示了联网AI的脆弱性——在复杂的真实环境下，能力与可信度之间存在巨大的鸿沟。

两者合在一起传递了一个清晰的信息：**下一代AI竞争力的核心，不是模型有多大，而是环境有多好**。
