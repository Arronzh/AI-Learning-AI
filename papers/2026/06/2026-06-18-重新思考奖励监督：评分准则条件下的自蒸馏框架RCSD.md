# 重新思考奖励监督：评分准则条件下的自蒸馏，让推理模型获得比 GRPO 更精细的反馈信号

📌 **核心摘要**
> 耶鲁大学团队提出 Rubric-Conditioned Self-Distillation（RCSD），一种将评分准则（rubric）作为结构化、细粒度监督信号，用于推理语言模型后训练的新框架。RCSD 通过两阶段管道（先学习生成任务专属评分准则，再训练准则指导的推理器），在多样化的科学推理基准上平均超越 GRPO 1.0 分、超越 OPSD 0.9 分。代码已开源：https://github.com/carriegu0818/RCSD

---

## 一、研究背景与动机

当前推理语言模型（如 DeepSeek-R1、Qwen 等）的后训练主要依赖两条技术路线：**监督蒸馏** 和 **基于可验证奖励的强化学习**。

**蒸馏路线** 的问题在于依赖昂贵且可能有噪声的思维链（Chain-of-Thought）标注。即使最终答案正确，中间推理步骤也可能不完美，而模型仍然要被迫模仿一条固定的推理轨迹。

**强化学习路线**（以 GRPO 为代表）将评估反馈压缩为一个标量信号——无论模型做了 100 步推理还是 5 步推理，最终只得到一个"对"或"错"的分数。这种粗粒度的反馈掩盖了回答中哪些方面可以改进、哪些步骤是正确的，形成 **信用分配瓶颈**。

研究人员指出："GRPO 在可验证领域（如数学、代码）效果显著，但在需要更丰富评判的非可验证领域，标量奖励提供的信息极其有限。"

针对这一困境，一种自然的改进思路是引入 **评分准则（rubric）**——将回答质量分解为多个显式标准，而非整体打分。然而，现有工作通常只是将 rubric 评分聚合为一个标量后送入 RL，这一做法实质上丢弃了 rubric 中蕴含的丰富文本反馈。

## 二、核心方法

### 何为 RCSD？

RCSD（Rubric-Conditioned Self-Distillation）的核心洞察是：**评分准则不应只在生成后用于打分，而应直接塑造优化过程中的 Token 级学习信号。**

### 两阶段管道

**阶段 I：学习评分准则生成器（Rubric Generator）**

- **学生策略**：仅根据问题生成 rubric
- **教师策略**：同时访问问题和参考答案，生成更高质量的 rubric
- 通过 **在策略自蒸馏（on-policy self-distillation）** 让学生匹配教师在学生自己采样轨迹上的 Token 分布

**阶段 II：评分准则指导的推理（Rubric-Conditioned Reasoning）**

- **学生策略**：仅根据问题生成推理轨迹
- **教师策略**：同时访问问题和阶段 I 生成的 rubric，提供密度更高的 Token 级引导
- 优化目标：最小化教师与学生沿学生自身轨迹的 KL 散度

### 与现有方法的本质区别

| 方法 | 监督信号 | 反馈粒度 | 对推理路径的约束 |
|------|---------|---------|----------------|
| RL (GRPO) | 标量奖励 | 序列级（稀疏） | 无显式约束 |
| OPSD | 参考答案轨迹 | Token 级（密集但路径特异） | 强制靠近单一路径 |
| **RCSD（本工作）** | **评分准则** | **Token 级 + 准则感知** | **允许多条有效路径** |

关键差异在于：OPSD 迫使模型模仿一条"标准答案"的路径，而 RCSD 则告诉模型"一个好的回答应该满足什么标准"，保留了多种有效推理路径的可能性。

> "Rubric 不是评分员，而是结构化监督接口。"——论文核心主张。

## 三、实验结果

### 主实验结果

RCSD 在 Qwen3-8B 骨干模型上表现亮眼：

- **总体平均分 70.6**，超越 GRPO 1.4 分、OPSD 0.9 分
- **RubricHub** 上提升最大，较基座模型提升 4.9 分
- **ResearchQA** 提升 8.2 分
- **SciBench** 达到 70.8，展现科学推理任务上的显著优势

值得注意的是，RCSD 仅用 **100 步训练** 就达到了这些结果，而 GRPO 需要 500 步。

### 跨领域泛化

在医学领域基准（MedMCQA、PubMedQA）上，RCSD 虽然主要在科学推理数据上训练，但仍保持了竞争性能，没有出现灾难性遗忘。

### Rubric 质量的影响

实验还显示，**学习到的 rubric 可以接近人工参考 rubric 的效果**，这意味着该框架在实践中具有可扩展性——无需为每个问题人工设计评分准则。

### 定性分析

作者观察到，OPSD 训练出的模型常常产生冗长推理链——反复重新计算已得出的中间量、在没有新信息的情况下修改之前的步骤。RCSD 则更加高效，因为准则级反馈让模型知道"哪一步出错了"，而非"整个都需要重来"。

## 四、技术启示与发展方向

**1. 从"标量奖励"到"结构化反馈"的范式转变**

RCSD 标志着后训练优化方式的一个重要方向：不再将评估信息压缩为标量，而是保留其结构。这对于数学验证以外的广泛领域（如科学推理、开放式问答、创意写作）尤为关键。

**2. On-Policy 与 Criterion-Aware 的结合**

它兼具两个优势：通过在策略蒸馏解决了 Off-Policy 的分布不匹配问题；通过准则感知解决了标量奖励的信息瓶颈问题。

**3. 现实意义：数据效率与扩展性**

RCSD 仅需约 10k 样本训练 rubric 生成器、30k 样本训练推理器——远小于许多大规模强化学习管道的需求。对于预算有限的学术团队而言，这是一个非常有吸引力的选择。

**4. 未来方向**

- 扩展到连续概率场景（如 Giry monad 的神经表示）
- 探索更复杂的准则结构（如分层准则、交互式准则）
- 与推理时计算缩放（Inference-Time Compute Scaling）结合

## 五、总结

RCSD 提出了一种全新的视角：**评分准则不应被浪费**。它将原本用于评估的准则转化为训练中的结构化监督信号，在 GRPO 和 OPSD 之间找到了更优的平衡点——既有 OPSD 的 Token 级密度，又有比标量奖励更丰富的信用分配能力。
